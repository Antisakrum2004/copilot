# 09 — Следующие шаги

## Приоритет 0 — Горящие баги (сейчас ломается)

- [ ] **Добавить `app.on('before-quit', cleanup)` в index.ts** — без этого аудио-захват и стриминг не останавливаются корректно при выходе. ffmpeg-процессы могут остаться zombie.
- [ ] **Удалить дублирующую отправку в broadcastTranscription** — sttService.ts отправляет транскрипцию дважды: через getWindow() и через BrowserWindow.getAllWindows(). Второе включает первое — убрать первое.
- [ ] **Проверить externalizeDepsPlugin() для win-audio-capture** — нативный модуль может не скомпилироваться при electron-vite build. Возможно нужно добавить в external list явно.

## Приоритет 1 — Стабильность

- [ ] **Retry-механизм для STT API** — при ошибке сети или 429/500 от Groq, повторить отправку чанка 1-2 раза с exponential backoff. Сейчас чанк просто теряется.
- [ ] **Автоматический перезапуск ffmpeg** — при падении ffmpeg child_process (проверка каждые 5с) перезапускать захват. Сейчас захват молча прекращается.
- [ ] **Валидация API ключей** — при сохранении в SettingsPanel делать тестовый запрос к Groq/OpenAI/OpenRouter и показывать результат (валидный/невалидный).
- [ ] **Определение реального sample rate WASAPI** — не захардкоживать 44100Hz, а получать из нативного модуля или системы. На некоторых машинах 48000Hz или 96000Hz.
- [ ] **Обработка edge-case: нет окон** — getWindow() может вернуть null если окна ещё не созданы или уже уничтожены. Добавить проверки в broadcast функции.

## Приоритет 2 — Фикс багов

- [ ] **Atomic сохранение настроек** — вместо пошагового `settings.save()` для каждого ключа, сохранять все настройки одним вызовом. Либо добавить транзакцию.
- [ ] **Индикатор обрезки подсказки** — если max_tokens=1024 не хватило, показать «...» или «Ответ обрезан» в SuggestionPanel.
- [ ] **Очистка suggestionHistory** — при сбросе транскрипции (transcription-clear) нужно также очистить историю подсказок в openrouterService.
- [ ] **Удалить pkg/ директорию или синхронизировать** — два package.json (src/ и pkg/) с разными зависимостями — источник путаницы. Привести к одному.

## Приоритет 3 — Новые фичи

- [ ] **Markdown-рендеринг в SuggestionPanel** — подключить react-markdown (или marked + sanitize) для рендеринга Markdown-подсказок. Сейчас отображается как plain text.
- [ ] **Глобальные горячие клавиши** — Ctrl+Shift+R для старта/останова записи, Ctrl+Shift+S для ручной подсказки, Ctrl+Shift+H для скрытия/показа overlay.
- [ ] **Системный трей** — иконка в трее с меню: Старт/Стоп, Показать/Скрыть, Настройки, Выход. Сворачивание в трей вместо закрытия.
- [ ] **Редактор системного промпта** — в настройках добавить textarea для изменения промпта 1С. Сохранять в electron-store, подгружать при стриминге.
- [ ] **Настройка параметров LLM** — temperature, max_tokens, auto-suggestion delay — вынести в UI настроек.
- [ ] **VAD (Voice Activity Detection)** — пропускать чанки с тишиной, не отправлять на Whisper API. Сэкономит rate limit и деньги.
- [ ] **Список аудио-устройств** — показать доступные микрофоны и динамики в настройках, позволить выбрать устройство.
- [ ] **Диагностическая панель** — показать: статус захвата, длину STT-очереди, количество API-вызовов, время последней транскрипции, статус стриминга.

## Приоритет 4 — Чистка / рефакторинг

- [ ] **Вынести inline-стили в CSS Modules или отдельные файлы** — убрать `<style>` теги из компонентов, использовать CSS Modules или отдельные .css файлы.
- [ ] **TypeScript strict mode** — включить `strict: true` в tsconfig, исправить неявные any и null-ошибки.
- [ ] **Единый логгер** — заменить console.log на структурированный логгер с уровнями (debug, info, warn, error) и записью в файл.
- [ ] **Рефакторинг IPC** — часть каналов не следует конвенции именования. Унифицировать: `domain:action` для всех.
- [ ] **Удалить неиспользуемые IPC-каналы** — `audio:micChunk`, `audio:speakerChunk`, `audio-loopback-data` не используются renderer-ом напрямую. Оценить необходимость.
